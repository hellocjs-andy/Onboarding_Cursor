# 方舟达富 H5 项目技术架构文档

## 一、项目概述

### 技术栈
- **构建工具**: Vite 4.5.2
- **前端框架**: React 18.2.0
- **开发语言**: TypeScript 5.2.2
- **状态管理**: Redux Toolkit 2.2.1
- **路由方案**: React Router DOM 6.22.3 (Hash 模式)
- **UI 组件库**: antd-mobile 5.34.0
- **HTTP 请求**: Axios 1.6.7
- **国际化**: i18next + react-i18next
- **图表库**: @antv/f2 3.8.9
- **样式方案**: Less + TailwindCSS + PostCSS
- **响应式方案**: postcss-pxtorem (REM 适配)

## 二、核心技术架构

### 2.1 路由架构

**路由模式**: HashRouter

**路由生成方式**: 基于文件系统的路由自动生成

**路由规则**:
- `pages/` 目录下的文件自动映射为路由
- `index.tsx` 映射为根路径 `/`
- 子目录结构映射为嵌套路径 (如 `asset/home/index.tsx` → `/asset/home`)
- 自动过滤 `hooks`、`components` 目录
- 自动过滤 `_` 开头的目录
- 自动过滤 `model` 文件和 `use` 开头的 hooks

### 2.2 样式方案

#### 2.2.1 多方案组合

```
Less (预处理器)
  +
TailwindCSS (原子化 CSS)
  +
PostCSS (自动转 REM)
  +
antd-mobile 组件库主题
```

#### 2.2.2 REM 适配方案

**基准**: 设计稿 750px，根字体大小 100px (即 1rem = 100px)

**PostCSS 配置**:
```javascript
'postcss-pxtorem': {
  rootValue: 100,
  propList: ['*'],
  exclude: /node_module/
}
```

## 三、开发规范

### 3.1 目录命名规范

- 页面目录：小驼峰 (如 `directClub/`)
- 组件目录：大驼峰 (如 `DurationTabs/`)
- 文件名：小写 + 连字符 (如 `env-config.ts`)
- React 组件：`.tsx`
- 工具函数：`.ts`

### 3.2 路由命名规范

- 与目录结构保持一致
- 全部使用小写
- 单词间用斜杠分隔
- 示例：`/asset/home`、`/trade/buy`

### 3.3 国际化文案规范

```typescript
// 带注释的翻译
i18next.t('common.confirm' /*确认*/);

// 文件结构
locales/
├── module/
│   ├── en.ts
│   ├── zh-CN.ts
│   └── zh-TW.ts
```

## 四、扩展指南

### 4.1 新增页面

1. 在 `src/pages/` 下创建目录和组件
2. 自动获得路由，无需手动配置
3. 如需 Redux 状态，在 `src/slices/` 创建 slice

**示例**:
```
src/pages/notification/
├── index.tsx          # 主页面
├── components/        # 页面级组件
└── hooks/            # 页面级 hooks
```

### 4.2 新增公共组件

1. 在 `src/components/` 创建组件目录
2. 遵循大驼峰命名
3. 提供 index.tsx 统一导出

**示例**:
```
src/components/CustomModal/
├── index.module.less  # 样式
└── index.tsx          # 组件
```

### 4.3 新增国际化文案

1. 在对应模块的翻译文件中添加
2. 三个语言文件同步更新

**示例**:
```typescript
// locales/product/en.ts
export default {
  new_feature: 'New Feature'
};

// locales/product/zh-CN.ts
export default {
  new_feature: '新功能'
};

// locales/product/zh-TW.ts
export default {
  new_feature: '新功能'
};
```
